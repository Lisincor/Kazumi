## 如何使用本教程

### 1. 带着问题读

每份文档都有"学习目标"，先记住这个问题：
- 读完要回答：这个模块解决了什么问题？
- 读完要能做：能用这个模块实现什么功能？

### 2. 对照源码读

教程会引用具体的代码文件和行号：
```
📁 lib/pages/popular/popular_controller.dart:1-10
```
表示你需要打开这个文件，对照第 1-10 行阅读。

### 3. 亲手改代码

教程中会有"小练习"：
```
🛠️ 小练习：
尝试在 PopularController 中添加一个新状态 `isEmpty`，
当列表为空时显示空状态提示。
```

### 4. 理解而不是背

遇到不理解的概念：
1. 搜索引擎搜索"Dart [概念名]"
2. 搜索 Flutter 官方文档
3. 在代码中搜索类似用法

## 项目核心架构

```
┌─────────────────────────────────────────────────┐
│                   UI 层                          │
│   pages/ (Page 组件 + Controller + Module)        │
└─────────────────────────────────────────────────┘
                      ↓ ↑
┌─────────────────────────────────────────────────┐
│                状态管理层                        │
│   MobX Store (响应式状态)                          │
└─────────────────────────────────────────────────┘
                      ↓ ↑
┌─────────────────────────────────────────────────┐
│                Repository 层                     │
│   数据访问接口抽象                                 │
└─────────────────────────────────────────────────┘
                      ↓ ↑
┌─────────────────────────────────────────────────┐
│                工具层                            │
│   GStorage (Hive) + Request (Dio)                │
└─────────────────────────────────────────────────┘
```

## 常见概念对照表

| 概念 | 类比理解 |
|------|---------|
| `Observable` | Vue 的响应式数据 |
| `Store` | React 的 useState |
| `Action` | Vuex 的 mutations |
| `Box` | Hive 的数据库表 |
| `Module` | Flutter Modular 的路由模块 |
| `Plugin` | 规则配置，定义如何抓取网站 |
